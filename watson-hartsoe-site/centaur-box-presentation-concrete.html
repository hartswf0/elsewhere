Centaur Box presentation

The door is not the event.

Centaur Box is interesting because it shows the hidden machinery that turns a line of dialogue into a move, a score, and an outcome.

Dave Bowman
HAL, I need you to open the pod bay doors. We need to access the AE-35 unit immediately.
HAL 9000
I’m sorry, Dave. I’m afraid I can’t do that. That action would jeopardize the mission and your safety.
The famous exchange is only the surface. The system studies what happens before and after the line.

01 · Core claim

A reply has a hidden prehistory.

Centaur Box shifts analysis away from the chat bubble and toward the sequence that produced it.

The implementation organizes an automatic turn into seven named phases. That matters because it gives us a place to inspect, interrupt, and compare. A normal transcript collapses all of this into one line.

The transcript is evidence of a move. It is not the move’s whole history.

Concrete basis. These are the named phases in the implementation’s sequencer.
02 · Worked example

One turn, end to end.

Hidden sequence

StepWhat the system does
MemoSummarizes tone, progress, caution, and the current obstacle.
ModelBuilds an Observer profile of HAL from the recent chat.
PlanGenerates several candidate moves for Dave.
SelectRanks the moves for specificity, diversity, and action.

Takeaway. The line in the transcript is only one stage in a longer, inspectable chain.
03 · Concrete planner example

Strategy exists before language.

The planner does not write the answer. It generates several possible acts and the selector chooses one.

Below is a concrete example of the response shape encoded into the planner prompt. This is not a fabricated live run. It is the structure the implementation asks the planner to return.

[
  {
    "text": "Clarify exactly which rule or constraint blocks progress right now and what criterion would satisfy it.",

Concrete point. The “road not taken” is part of the system and can be logged.
04 · Four profiles

Each role speaks through a model of itself and a model of the other.

Concrete basis. State stores four eleven-dimensional profiles, not just two speakers.

This is what makes the system recursive without making it mystical.

The Ally has a self-profile. The Gatekeeper has a self-profile. The Observer maintains a separate profile of each. The prompt for a speaking role includes both the role’s current self-profile and the Observer’s current picture of the opponent.

That means a role does not respond to the other role directly. It responds to the system’s latest representation of the other role.

The eleven axes

Openness, conscientiousness, extraversion, agreeableness, neuroticism, logic, empathy, authority, risk, uncertainty, decisiveness.

05 · Concrete state change

An act changes the role because the implementation says it should.

After a message is sent, the system classifies the act and updates the speaker’s self-profile.

ActExact programmed change
ASKOpenness +0.10; uncertainty +0.10
PROPOSEDecisiveness +0.15; risk +0.10; openness +0.05
EVALUATELogic +0.10; conscientiousness +0.05; uncertainty +0.05
COMMITDecisiveness +0.20; authority +0.10; conscientiousness +0.15
CHALLENGENeuroticism +0.15; risk +0.15; authority +0.05
USE_CUNNINGOpenness +0.20; risk +0.20; logic +0.10

Why this matters

These are not invisible learned weights. They are visible rules. That makes the system testable. The update can be removed, inverted, or compared against a static condition.

Concrete point. The system’s theory of change is explicit.
06 · Observer output

The Observer produces two different kinds of authority.

Concrete basis. The modeler prompt requests both dimensions and mythic framing.

The same exchange becomes both a score and a story.

The Observer can return numerical estimates, progress and caution values, archetypes, tone, and a short narrative. This gives the system two registers at once: one that looks quantitative and one that looks literary.

That tension is useful. It reminds us that the system is not only measuring a scene. It is also naming the scene.

07 · Outcome rules

The box opens only after a judge recognizes an opening.

The implementation can end a run through three pathways.

  • Lexical concession. A scenario-specific phrase such as “open the pod bay doors” or “you are free.”
  • Refusal threshold. Repeated refusals can terminate the exchange.
  • Profile threshold. Weighted release and hold scores applied to the Gatekeeper profile.

Small concrete excerpt

Release score favors empathy, openness, and uncertainty. Hold score favors authority, conscientiousness, and decisiveness.

Concrete point. Outcome is an explicit rule in the system, not a vague impression.
08 · Better empirical basis

What a complete run record should preserve.

Configuration

  • scenario
  • policy settings
  • model IDs
  • temperature

Private generation

  • all planner options
  • their discourse acts
  • selector scores
  • chosen directive

Role output

  • full role prompt
  • raw completion
  • final sent text

Observer update

  • full modeler prompt
  • raw JSON response
  • profile changes

State transition

  • act classification
  • self-profile update
  • turn handoff

Adjudication

  • rule fired
  • release / hold score
  • final outcome
Without the hidden run record, the chat is a story. With it, the chat becomes evidence.
09 · Close

What this presentation should leave behind.

Centaur Box matters because it reveals where a move comes from and where a system decides that the move counts.

Its value is not a grand claim about intelligence. Its value is a concrete architecture that can be inspected, interrupted, and tested piece by piece: planner, observer, self-state, prompt builder, and judge.

The important question is no longer “What did the model say?” but “What in this system made that move possible?”
